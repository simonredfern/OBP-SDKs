# OBP SDK Strategy

## Source of Truth

The OBP Resource Docs / OpenAPI 3.1 spec is the single source of truth for all SDKs.

The pipeline is:

```
Resource Docs -> OpenAPI 3.1 JSON -> Auto-generated SDKs
                                  -> MCP Server (AI-native SDK)
                                  -> API Explorer / Swagger UI
```

No hand-written SDKs. All SDKs are auto-generated from the OpenAPI spec using OpenAPI Generator.

## Why This Approach

- In the age of AI agents, the OpenAPI spec itself is the primary interface. Agents read the spec and generate correct API calls on the fly.
- Auto-generated SDKs stay in sync with the API by definition — no drift.
- Maintenance cost is near zero once the pipeline is set up.
- The MCP server (`obp-mcp`) is effectively the SDK for AI agents.

## Investment Priority

The highest-leverage work is making the OpenAPI spec excellent:

1. Rich descriptions — not just field names, but meaning, constraints, and examples
2. Authentication documentation — clear auth flows and security schemes
3. Complete request/response examples
4. Accurate error schemas
5. Semantic grouping via tags

## Repository Structure

Mono repo: `github.com/OpenBankProject/OBP-SDKs`

```
OBP-SDKs/
  generator/           # OpenAPI Generator config, templates, CI
  openapi-spec/        # The generated spec (or fetched from OBP-API)
  python/
  java/
  csharp/
  typescript/
  README.md
```

### Why a mono repo

- SDKs are generated output — no external contributors editing them
- One CI job generates everything in one pass
- Versioning stays in sync — one tag means all SDKs are at the same API version
- Less repo sprawl

## CI Pipeline

On each OBP-API release:

1. Generate the OpenAPI 3.1 spec from Resource Docs
2. Run OpenAPI Generator for each target language
3. Commit generated code to OBP-SDKs
4. Tag a release
5. Publish to package registries:
   - `python/` -> PyPI
   - `typescript/` -> npm
   - `java/` -> Maven Central
   - `csharp/` -> NuGet

## Target Languages

- Python
- Java
- C# (.NET)
- TypeScript/JavaScript

More can be added trivially since OpenAPI Generator supports 50+ languages.

## For AI Agents

AI agents don't need traditional SDKs. They can:

- Read the OpenAPI spec directly and generate correct API calls
- Adapt to API changes by re-reading the spec
- Use the MCP server as a tool interface

The MCP server is the AI-native SDK.

## Tooling

- [OpenAPI Generator](https://openapi-generator.tech/) for SDK generation
- OpenAPI 3.1 spec generated by `OpenAPI31JSONFactory.scala` in OBP-API
